현재 작업 디렉토리 기준 `output/` 폴더의 CSV 파일을 읽어, 참고 문서로 활용할 수 있도록 구조화된 마크다운 파일로 요약합니다.

## 단계

1. `output/` 폴더(CWD 기준)의 모든 CSV 파일을 읽습니다. $ARGUMENTS에 파일명이 지정된 경우 해당 파일만 처리합니다.
2. 모든 행을 사용합니다 — `item_type == "review"`와 `item_type == "reply"` 모두 포함. 답글에는 유용한 Q&A 맥락(예: 관리사무소 답변, 추가 정보)이 담겨 있는 경우가 많습니다. 헤더에 리뷰/답글 수를 각각 기록하세요.
3. 아래 주제별로 리뷰를 분석·요약합니다. 각 주제마다 다음 항목을 모두 포함한 상세 요약을 작성하세요:
   - **sentiment**: 긍정 / 부정 / 혼재 / 중립 / 언급없음
   - **긍부정 비율**: 긍정:부정 비율 (예: "긍정 70% / 부정 30%"). 언급 없으면 "해당 없음"
   - **언급 수**: 관련 언급의 대략적인 수 (전체 리뷰 대비 비율도 표시, 예: "약 15건 (전체의 ~10%)")
   - **요약**: 전체적인 패턴을 5–8문장으로 서술. 다수 의견과 소수 의견을 모두 포함하고, 소수 의견은 "일부는 ~" 또는 "반면 ~"으로 명시
   - **소수의견 / 반론**: 다수 의견에 반하는 의견이 있으면 반드시 명시. 없으면 "없음"
   - **세부 조건**: 동(棟), 층(저층/중층/고층), 평형(타입), 향(남향/동향 등), 시기별로 의견이 갈리면 반드시 분리 명시. 차이 없으면 "동/층/타입별 차이 없음"으로 명시
   - **팩트**: 수치, 날짜, 스펙 등 객관적 사실만 모음 (예: "엘베 교체 예정 2028년", "분리수거 토~일"). 없으면 "없음"
   - **대표 인용**: 긍정·부정 각각 최소 1개씩, 총 3–5개의 실제 원문 인용 (따옴표 사용). 해당 없으면 생략 가능

### 요약 주제
- 층간소음 / 벽간소음
- 주차
- 관리비
- 관리사무소 / 경비
- 엘리베이터
- 누수 / 결로 / 하자
- 방음 / 환기
- 커뮤니티시설
- 학군
- 교통 / 입지
- 편의시설 (마트, 병원 등)
- 이웃 / 주민 분위기
- 재건축 / 재개발 가능성
- 가격 / 실거래가 언급
- 일조 / 조망 / 향
- 세대 수 / 단지 규모
- 주변 혐오시설 (유흥가, 공장, 소음원 등)
- 인터넷 / 통신 / 스마트홈
- 분리수거 / 쓰레기 처리
- 반려동물
- 택배 / 무인택배함
- 크기 / 평면 / 구조 / 침대, 냉장고, 워시타워, 세탁기, 건조기 등등 가전가구 크기 정보 (상세하게 적어줄 것)
- 기타 특이사항

4. 마지막에 **팩트 요약** 섹션을 별도로 추가: 리뷰 전체에서 수집된 객관적 수치/날짜/스펙을 한 곳에 모아 bullet로 정리
5. Write 도구를 사용해 요약 결과를 `.claude/skills/apt-review/reference/{아파트명}.md` (CWD 기준)에 저장합니다.

## 출력 형식

```markdown
# {아파트명} 입주민 리뷰 요약
- 수집일: {크롤링 날짜 or 오늘 날짜}
- 총 리뷰 수: N개 (review: N개, reply: N개)

## 층간소음 / 벽간소음
- sentiment: 부정
- 긍부정 비율: 긍정 20% / 부정 80%
- 언급 수: 약 N건 (전체의 ~X%)
- 요약: ... (다수 의견 + 소수 의견 모두 포함)
- 소수의견 / 반론: ...
- 세부 조건: 201·202동은 소음 있음, 안쪽 동은 조용 / 저층보다 중층이 더 심하다는 의견 있음
- 팩트: 없음
- 대표 인용: "..." (부정) / "..." (긍정) / "..."

## 주차
...
```
